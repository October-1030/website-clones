# Storybound 1.13.1 原版提示词流水线

## 数据来源

提示词由本机已安装的 Storybound 1.13.1 客户端前端资源提取。生成脚本位于 `scripts/extract-original-prompt-templates.mjs`，结构化产物为 `desktop-app/original-prompt-library.json`。产物只包含客户端随包分发的模板与运行配置，不包含 API Key、Cookie、账户或激活数据。

## 步骤映射

| 本地步骤 | 原版模板与逻辑 |
| --- | --- |
| 文案预审 | 全局预审模板 |
| 智能改写与封面元数据 | 先执行全局 Writer Agent + 当前赛道改写模板并保存自评，再独立执行当前赛道元数据模板，保留 title/subtitle/summary/tags/comments 五字段 |
| 影视分镜分句 | Storyboard Agent 按原版尾部锚点法切分并校验全文覆盖；人物类赛道再生成人物一致性卡 |
| 生成绘图提示词 | Producer Agent + 当前赛道绘图模板 + 当前画风 prefix/suffix/negativePrompt |
| 批量生图 | MiniMax image-01，原版默认 9:16、并发 3；当前任务模型支持最多 60 镜。早期提取到的 18 镜是特定默认请求上限，不是产品硬上限；原作者真实任务截图已出现 33/33 镜。失败时按赛道 L2/L3 备用场景重试 |
| TTS 配音 | 旁白任务逐镜使用当前 MiniMax 或火山配置；播客严格按 `[A]`/`[B]` 使用两种音色；外部音频读取真实时长后生成字幕时间线 |

## 原版映射

共 8 个赛道：人物故事、健康图书、传统文化、绘本故事、电商带货、心灵鸡汤、民间故事、通用故事。共 13 套画风：黑白摄影、写实彩色、油画风格、现代电影、古风电影、复古胶片、水彩治愈、杂志插画、皮克斯 3D、中国水墨、民间故事工笔风、吉卜力、黑板橙绘。

赛道切换时恢复原版默认画风；生图前可在任务结果区检查并编辑最终提交给 MiniMax 的完整提示词和负面提示词。人物参考图以本地 base64 `subject_reference` 发送，图片、音频、封面和草稿均写入任务独立目录。

## 凭据边界

本地服务优先读取用户配置的 LLM 凭据；未单独配置 LLM 时，使用 `minimax-secrets.txt` 中同一份 MiniMax API Key 调用 OpenAI 兼容的文本接口。凭据只在 Node 服务端读取，不返回浏览器，也不写入仓库。
