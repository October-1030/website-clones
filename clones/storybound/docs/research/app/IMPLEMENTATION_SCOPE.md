# 独立复刻范围与实施建议

## 1. 可以完整独立实现

- 桌面导航、所有创作表单、任务详情、队列、历史、模板和实验室 UI。
- 图文 7 步状态机，以及全自动/半自动/直接出片三种起点。
- 暂停确认、编辑中间产物、断点续跑、局部修复、从指定步骤重跑。
- 本地 SQLite、任务文件夹、事件日志、批次执行和状态恢复。
- 用户自带 LLM、图片、TTS、IMA 凭据的 provider 适配层。
- 本地 SenseVoice ASR、FFmpeg 音频处理、字幕生成与时间轴对齐。
- HTML 动画场景渲染、预览和 MP4 合成。
- 人物素材库、提示词模板、草稿模板、画图和配音实验室。
- 独立实现剪映草稿打包器，前提是持续用实际剪映版本做兼容测试。

## 2. 必须替换或降级的部分

| 原功能 | 独立实现方案 |
| --- | --- |
| Storybound 激活/配额/积分 | 改为本地开源版无激活，或接我们自己的账户和计费系统 |
| “全能绘图”私有计费接口 | 改为 OpenAI 兼容、魔搭、RunningHub 或其他用户自带 API |
| 创作市场后台 | 第一阶段做本地模板库；需要多人交易时再设计独立后台 |
| 视频号监控/解析后台 | 第一阶段保留手动导入视频和 ASR；后续使用合规的数据源适配器 |
| 官方更新与公告 | 使用我们自己的版本发布和更新源 |
| Storybound 1.13.1 客户端内置提示词 | 从用户提供并已安装的客户端包中结构化提取，按原赛道、画风和步骤映射接入本地流水线；不包含原站凭据与授权服务 |

## 3. 推荐目录

保持现有落地页克隆不动，在同一目标目录下新增独立桌面项目：

```text
clones/storybound/
  src/                    # 已完成的官网落地页克隆
  desktop-app/            # 后续新增的 Tauri 桌面产品复刻
  docs/research/app/      # 本次产品研究
```

## 4. 推荐实施顺序

### 阶段 A：可操作的桌面壳

- 完成侧边栏、创建首页和全部路由。
- 用本地 SQLite 跑通任务、队列、历史、模板数据。
- 先用模拟 provider 验证三种模式和全部状态转换。

### 阶段 B：图文任务闭环

- 接入用户自带 LLM。
- 实现 7 步流水线、暂停/重跑/修复和任务产物目录。
- 接入一种图片 provider 与一种 TTS provider。
- 实现字幕、音频切分和基础剪映草稿导出。

### 阶段 C：HTML 动画与音乐 MV

- HTML 场景模板、离屏渲染、逐帧截图和 FFmpeg 合成。
- 音乐 MV 的歌词分组、生图、时间轴对齐和草稿打包。
- 再补动态视频、声音克隆和高级模板。

### 阶段 D：外围工作台

- 人物素材、画图/配音实验室、选品助手。
- 最后再评估对标监控、市场和多用户后台，因为这三块外部依赖和合规成本最高。

## 5. 验收标准

- 三种图文模式在相同输入下能走到正确起始步骤。
- 任一步中断后重启应用，仍能检测已有产物并从正确位置恢复。
- 修改 Step 1/2/3 产物不会重跑更早步骤。
- 单条失败可修复，不覆盖成功图片或音频。
- 批量任务严格串行，当前条暂停/取消不会破坏后续队列。
- HTML 与 MV 在替换图片、字幕、音色或模板后可以只重打包。
- 所有外部凭据都进入系统钥匙串，日志和导出文件不泄露密钥。
